PROBLEMA ORIGINAL:
  ✗ Timeouts insuficientes (30s) → navegación a SPAs lentas fallaba
  ✗ Sin reintentos automáticos → había que reintentar manualmente
  ✗ Errores crípticos sin contexto → no sabías por qué fallaba (DNS? URL? sesión?)
  ✗ Detectores de carga débiles → tablas medio cargadas se daban como "listas"

  ✓ Timeouts mayores y configurables (30s → 50s)
  ✓ Retry automático inteligente: 3 intentos con backoff (1.5s, 3s, 6s)
  ✓ Análisis de errores específicos: DNS vs conexión vs timeout vs ruta vs sesión
  ✓ Auto-detectores de carga: spinner + "Cargando" + DOM quieto
  ✓ Log diagnóstico completo: ~/.../diagnostico.log con timestamps + símbolos

  4. GET /tabla                       ← datos COMPLETOS

DIAGNÓSTICOS ESPECÍFICOS:
  DNS_FAILED          → DNS no resolvió (se repara automáticamente)
  NETWORK_FAILED      → Sin conexión de red
  CONNECTION_FAILED   → Servidor rechazó / no responde
  TIMEOUT             → Página tardó > 50s
  NOT_FOUND (404)     → URL no existe
  FORBIDDEN (403)     → Sesión expirada / rol insuficiente
  SESSION_EXPIRED     → Redirigió a login
  BROWSER_ERROR       → Navegador crasheó

LOG DIAGNÓSTICO:
  Ruta: ~/nexus/conector-navegador/diagnostico.log
  Tiempo real: tail -f diagnostico.log
  Formato: [timestamp ISO] operación: detalles
           [timestamp] → sub-acción
           [timestamp] ✓ éxito

Autenticación / Token de acceso

Reservo (este plan) usa autenticación por sesión (Django), no API-key self-service. El "token" es la cookie de sesión:

Token (sessionid): 7ea50ieybn069534vd54zv77b3yz03yy
csrftoken: w7YCDUllbXiTeOeUFFzzzAggls3GmSnk0Nj3byezi9SEosQjEicV8KfHpc5uvoxZ

⚠️ Sensible: este token da acceso a tu cuenta. No lo compartas. Caduca con el tiempo o al cerrar sesión; se renueva volviendo a loguear en POST /accounts/login/ (username, password, csrfmiddlewaretoken).

Uso: mandá la cookie en cada request:
curl "https://reservo.cl/appointment/viewAppt/" -H "Cookie: sessionid=<token>; csrftoken=<csrf>"
Para POST agregá -H "X-CSRFToken: <csrf>" y -H "X-Requested-With: XMLHttpRequest".
